# HelpMeAI Coding Standard

## General Rules

- Write clean and readable code.
- Keep functions small and focused.
- Avoid duplicate code.
- Use meaningful variable names.
- Every module should have comments where necessary.
- Never hardcode secrets or API keys.
- Validate all user inputs.
- Handle all errors gracefully.

---

## Security Rules

- Follow the Principle of Least Privilege.
- Use HTTPS everywhere.
- Encrypt sensitive data.
- Sanitize all inputs.
- Log security events.

---

## Git Rules

- One feature per commit.
- Clear commit messages.
- Never push broken code.
- Review before merge.

---

## AI Development Rules

- AI decisions must be traceable.
- AI outputs should be logged where appropriate.
- Human approval required for critical actions.
- AI should fail safely.
